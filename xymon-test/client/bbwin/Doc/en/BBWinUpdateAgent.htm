BBWinUpdate

Description

BBWinUpdate agent is an agent used to update the local BBWin configuration from the central hobbit server. It use the hobbit "config" protocol word. This agent can be used with hobbit 4.1 or higher. The bbdisplay used with the agent is the first listed in the bbwin section of the BBWin.cfg configuration file.

Configuration

  • setting directive syntax : 

    <setting name="" value="" />

    name value
    filename This is the filename that will be downloaded from the hobbit server. You have to upload the files on the hobbit server ( Location : $HOBBITHOME/server/etc). You can specify several filename settings so you can play with multiple configuration files on the hobbit server.You can use environment variables in the value. Example : "bbwin\%computername%.cfg"

Examples

  • This an example of configuration of the agent : 

    <bbwinupdate>
	<setting name="filename" value="bbwin/bbwindefault.cfg" />
	<setting name="filename" value="bbwin/bbwinSql.cfg" />
</bbwinupdate>
  • This an example of configuration of stored on the hobbit server ($HOBBITHOME/server/etc/bbwin/bbwindefault.cfg for example): 

    <?xml version="1.0" encoding="us-ascii" ?>
<configuration>
<cpu>
	<!-- If true, the agent  will always report with green status -->
	<setting name="alwaysgreen" value="false" />
	<setting name="default" warnlevel="90" paniclevel="95" delay="3" />
</cpu>
<disk>
	<setting name="default" warnlevel="85%" paniclevel="95%" />
	<setting name="cdrom" value="true" />
</disk>
<msgs>
	<setting name="alwaysgreen" value="false" />
	<setting name="summary" value="true" />
	<match logfile="System" type="error" delay="1h" alarmcolor="red" />
	<match logfile="System" type="warning" alarmcolor="yellow" />
	<match logfile="Application" type="error" delay="1h" alarmcolor="red" />
	<match logfile="Application" type="warning" alarmcolor="yellow" />
	<match logfile="Security" type="fail" />
	
	<!-- Some ignore rules examples 
	<ignore logfile="Application" eventid="1030" />
	<ignore logfile="System" source="W32Time" />
	-->
</msgs>
<uptime>
	<setting name="delay" value="30m" />
</uptime>
</configuration>

Notes

  • you can not upgrade the bbwin namespace by security
  • a good idea is to create a subdirectory bbwin in your hobbit etc folder
  • the agent is using the bbwin tmp folder to store the update file
